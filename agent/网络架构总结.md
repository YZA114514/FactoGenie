# 🎯 LayoutDQN 网络架构总结

## 📌 核心改进

根据状态空间设计 `s_t = {s_t1, s_t2, s_t3}`，实现了**三分支DQN网络**，包含以下核心组件：

### ✅ **1. CNN嵌入布局 s_t1: (H, W, N)**

```
输入: [batch, H, W, N] 三维张量
     最后一维是待放置物体的One-Hot编码
     
CNN: Conv2d(N→32) → Conv2d(32→64) → Conv2d(64→64)
     每层后接 ReLU + BatchNorm

输出: [batch, 64*H*W] 空间特征向量
```

**设计理由**:
- ✅ 三维表示能区分不同物体的位置
- ✅ 多层卷积捕捉局部→全局的空间模式
- ✅ BatchNorm提高训练稳定性

---

### ✅ **2. GCN嵌入物料流 s_t2: (N×N邻接矩阵)**

```
输入: [batch, N, N] 邻接矩阵
     A[i,j]=1 表示有从物体i到j的物料流
     
GCN: 节点嵌入 → GCN Layer 1 → ReLU → GCN Layer 2
     使用标准GCN公式: H' = σ(D^(-1/2) A D^(-1/2) H W)
     
池化: 全局平均池化

输出: [batch, 128] 图特征向量
```

**设计理由**:
- ✅ GCN自然处理图结构的物料流关系
- ✅ 2层GCN能捕捉2跳邻居的信息
- ✅ 全局池化聚合所有物体的关系特征

---

### ✅ **3. MLP嵌入待放置模块 s_t3: (N维One-Hot)**

```
输入: [batch, N] One-Hot编码向量
     表示当前要放置哪个模块
     
MLP: Linear(N→64) → ReLU → Linear(64→64) → ReLU

输出: [batch, 64] 模块特征向量
```

**设计理由**:
- ✅ 告诉网络当前要放置哪个模块
- ✅ 让网络根据模块特性做出针对性决策
- ✅ 完整实现状态空间 s_t = {s_t1, s_t2, s_t3}

---

### ✅ **4. 特征融合 + Dueling DQN**

```
融合: Concat([CNN特征, GCN特征, MLP特征])
     → [batch, 64*H*W + 128 + 64]

Dueling:
  ├─ 价值流: Fusion → FC(256) → FC(1) → V(s)
  └─ 优势流: Fusion → FC(256) → FC(num_actions) → A(s,a)
  
Q值: Q(s,a) = V(s) + (A(s,a) - mean(A(s,a)))
```

**设计理由**:
- ✅ 融合三分支特征（空间+关系+模块）
- ✅ Dueling架构提高学习效率
- ✅ 分离V和A使训练更稳定

---

### ✅ **5. 动作掩码输出**

```
掩码预测: Fusion → FC(128) → FC(num_actions) → Sigmoid
         → [batch, num_actions] (0-1之间)

应用掩码: Q值中无效动作设为 -1e9
```

**设计理由**:
- ✅ 自动学习物理约束（位置占用）
- ✅ 避免选择无效动作
- ✅ 可作为辅助训练信号

## 💻 快速使用

### **1. 创建网络**

```python
from agent.dqn_model import LayoutDQN

model = LayoutDQN(
    height=10,           # 工厂高度
    width=10,            # 工厂宽度
    num_objects=5,       # 物体数量
    num_actions=400,     # 动作数量
    use_dueling=True     # 使用Dueling架构
)
```

### **2. 准备输入**

```python
from agent.dqn_model import create_layout_tensor, create_material_flow_matrix
import torch

# 1. 布局 (s_t1)
placed = [(0,0,0), (1,2,3), (2,5,5)]
layout = create_layout_tensor(10, 10, 5, placed).unsqueeze(0)

# 2. 物料流 (s_t2)
flows = [(0,1), (1,2), (2,0)]
material_flow = create_material_flow_matrix(5, flows).unsqueeze(0)

# 3. 待放置模块 (s_t3) - 假设要放置物体3
current_object = torch.zeros(1, 5)
current_object[0, 3] = 1

# 4. 动作掩码
action_mask = torch.ones(1, 400)
action_mask[0, :50] = 0  # 前50个动作无效
```

### **3. 计算Q值**

```python
q_values, predicted_mask = model(layout, material_flow, current_object, action_mask)
```

### **4. 选择动作**

```python
# 贪心
action = model.get_action(layout, material_flow, current_object, action_mask, epsilon=0.0)

# epsilon-greedy
action = model.get_action(layout, material_flow, current_object, action_mask, epsilon=0.1)
```

## 🎯 与状态空间设计对应关系

| 状态/动作设计 | 实现方式 | 代码位置 |
|-------------|---------|---------|
| **s_t1: 布局状态 (H×W×N)** | 3层Conv2d处理One-Hot编码 | `self.cnn` |
| **s_t2: 运输状态 (N×N DAG)** | 2层GCN处理邻接矩阵 | `self.gcn` (包含`GraphConvLayer`) |
| **s_t3: 待放置模块 (N)** | 2层MLP处理One-Hot编码 | `self.object_mlp` |
| **特征融合** | Concat三分支特征 | `fused_features` |
| **动作空间 a_t = (x,y,θ)** | num_actions = H×W×4 | 网络输出维度 |
| **DQN输出** | Dueling架构输出Q值 | `self.value_stream` + `self.advantage_stream` |
| **动作掩码** | Sigmoid预测 + 应用到Q值 | `self.mask_predictor` |

## 🔧 可配置参数

```python
model = LayoutDQN(
    height=10,                        # 必需
    width=10,                         # 必需
    num_objects=5,                    # 必需
    num_actions=400,                  # 必需
    cnn_channels=[32, 64, 64],       # 可选: CNN通道数
    gcn_hidden_dim=64,               # 可选: GCN隐藏维度
    gcn_output_dim=128,              # 可选: GCN输出维度
    fusion_hidden_dim=256,           # 可选: 融合层维度
    use_dueling=True                 # 可选: 是否Dueling
)
```

**调参建议**:
- 小规模问题 (5×5): 减小所有维度 ÷2
- 大规模问题 (20×20): 增大所有维度 ×2
- 训练不稳定: 降低学习率，增大batch size
- 过拟合: 增加Dropout，减小网络容量
