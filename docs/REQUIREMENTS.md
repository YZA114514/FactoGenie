# FactoGenie 功能需求文档

## 项目概述

FactoGenie 是一个基于深度强化学习的工厂布局优化系统，帮助用户自动规划功能单元的最优摆放位置。

---

## 功能模块

### 1. 工厂配置输入

#### 1.1 手动输入
- 通过表单界面配置工厂参数
- 支持配置项：
  - **画布尺寸**：工厂区域的宽度和高度
  - **功能单元**：名称、尺寸、是否可移动、初始位置（可选）
  - **障碍物**：名称、尺寸、位置
  - **物料路线**：起点、终点、物料类型、批量大小、运输工具
  - **装配工艺**：工位、输入物料、输出物料、加工时间
  - **运输工具**：类型、数量、速度

#### 1.2 JSON文件上传
- 支持上传 `factory_config.json`（工厂配置）
- 支持上传 `layout_config.json`（布局配置）
- 上传后自动验证格式，显示错误提示

#### 1.3 配置文件格式
参考：
- `simulation/configs/chair_factory.json`
- `simulation/layouts/chair_layout.json`

### 2. 特殊约束设置

用户可设置额外的布局约束条件：

| 约束类型 | 说明 | 示例 |
|----------|------|------|
| **固定位置** | 指定单元必须放置在特定坐标 | 收货区固定在左侧入口 |
| **相邻约束** | 指定两个单元必须紧邻 | 工位1和工位2必须相邻 |
| **贴墙约束** | 指定单元必须靠墙放置 | 发货区必须靠右墙 |

```typescript
interface Constraints {
  fixed_positions: { unit_id, x, y, angle }[];
  adjacency: { unit_a, unit_b, direction }[];
  wall_attach: { unit_id, wall }[];  // wall: top/bottom/left/right
}
```

### 3. 训练参数配置

所有参数均可由用户设置，提供默认值：

#### 3.1 基础参数
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `total_steps` | 总训练步数 | 50000 |
| `learning_rate` | 学习率 | 0.00002 |
| `batch_size` | 批量大小 | 32 |

#### 3.2 经验回放
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `replay_size` | 回放缓冲区大小 | 50000 |
| `replay_start_size` | 开始训练前收集的样本数 | 5000 |

#### 3.3 探索参数
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `epsilon_start` | 初始探索率 | 1.0 |
| `epsilon_final` | 最终探索率 | 0.05 |
| `epsilon_decay_frames` | 探索率衰减步数 | 150000 |

#### 3.4 网络参数
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `sync_target_every` | 目标网络同步间隔 | 2000 |
| `double_dqn` | 是否使用Double DQN | false |
| `dueling` | 是否使用Dueling架构 | false |
| `noisy_net` | 是否使用Noisy Net | false |
| `prioritized` | 是否使用优先经验回放 | false |

#### 3.5 奖励权重
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `weight_distance` | 距离评分权重 | 0.25 |
| `weight_logistics` | 物流强度权重 | 0.25 |
| `weight_flow` | 流线清晰度权重 | 0.15 |
| `weight_throughput` | 产量权重 | 0.20 |
| `weight_utilization` | 利用率权重 | 0.15 |

> 注：所有权重之和必须等于 1.0

#### 3.6 摆放顺序
| 选项 | 说明 |
|------|------|
| `size_desc` | 按面积降序（默认） |
| `size_asc` | 按面积升序 |
| `flow_desc` | 按物流强度降序 |
| `random` | 随机顺序 |
| `process_flow` | 按工艺流程顺序 |
| `logistics_intensity` | 按物流强度顺序 |

#### 3.7 检查点参数
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `checkpoint_interval` | 每多少episode保存一次模型权重 | 1000 |

> 设为 0 表示只保存最终模型，不保存中间过程

#### 3.8 校准参数
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `calibrate_episodes` | 校准回合数（随机摆放次数） | 0（不校准） |
| `throughput_target` | 用户指定的吞吐量目标 | None（自动估计） |

> 校准流程：
> 1. 运行 N 次随机摆放，收集各项指标
> 2. 使用 P15 作为 best 边界，P90 作为 worst 边界
> 3. 结果缓存到 `data/calibrations/`，配置不变则复用
> 
> 建议首次训练时设置 `calibrate_episodes=100`，后续相同配置可设为 0

### 4. 训练记录保存

每次训练自动保存：

#### 4.1 配置信息
- 工厂配置 (`factory_config.json`)
- 布局配置 (`layout_config.json`)
- 约束条件 (`constraints.json`)
- 训练参数 (`training_params.json`)

#### 4.2 训练过程
- 奖励曲线 (`rewards.csv`)
- 损失曲线 (`losses.csv`)
- 各项指标 (`metrics.csv`)

#### 4.3 模型检查点
- 按 `checkpoint_interval` 间隔保存模型权重
- 自动标记最佳模型 (`model_best.pth`)
- 保存最终模型 (`model_final.pth`)

#### 4.4 布局快照
- 每个检查点对应的布局 (`layout_ep{N}.json`)
- 最佳布局 (`layout_best.json`)

#### 4.5 存储结构
```
data/results/{task_id}/
├── config/
│   ├── factory_config.json
│   ├── layout_config.json
│   ├── constraints.json
│   └── training_params.json
├── metrics/
│   ├── rewards.csv
│   ├── losses.csv
│   └── metrics.csv
├── checkpoints/
│   ├── model_ep1000.pth
│   ├── model_ep2000.pth
│   ├── model_best.pth
│   └── model_final.pth
├── layouts/
│   ├── layout_ep1000.json
│   ├── layout_ep2000.json
│   └── layout_best.json
└── heatmaps/           # 可选
    └── ep1000_step3.json
```

### 5. 训练进度显示

#### 5.1 实时进度
- 当前步数 / 总步数
- 当前 Episode
- 预计剩余时间
- 探索率 (ε)

#### 5.2 实时曲线
- 奖励曲线（每Episode更新）
- 损失曲线
- 各项指标曲线（距离、物流、产量、利用率、流线清晰度）

#### 5.3 控制操作
- 暂停 / 恢复
- 提前停止

---

### 6. 优化结果可视化

#### 6.1 布局渲染
- 2D俯视图显示功能单元布局
- 区分不同类型单元（工位、仓库、码头、障碍物）
- 显示物料流线（箭头连线）

#### 6.2 摆放过程回放
- 逐步展示每个功能单元的放置过程
- 可暂停、快进、后退
- 显示当前正在放置的单元信息

#### 6.3 决策依据可视化（动作热力图）
- 显示 Agent 对每个位置的 Q 值评估
- 热力图形式展示（颜色表示 Q 值高低）
- 支持切换不同旋转角度（0°, 90°, 180°, 270°）
- 高亮显示最终选择的位置

#### 6.4 历史记录对比
- 列表展示所有训练记录
- 支持查看任意检查点的布局
- 对比不同训练结果的指标

#### 6.5 结果导出
- 导出布局配置（JSON格式）
- 导出指标报告
