## 迁移步骤

把现有代码迁移到新结构：

```bash
# 1. 创建新目录结构
mkdir -p core/{agent,environment,simulation,tools}
mkdir -p webapp/{backend/{api/routes,schemas,services,db,workers},services,frontend}
mkdir -p data/{uploads/{cad,configs},layouts/{templates,experiments},results,exports}
mkdir -p docs/{api,guides,design}
mkdir -p scripts tests/{core,backend,frontend}
mkdir -p deploy/docker

# 2. 移动核心代码
mv agent/* core/agent/
mv environment/* core/environment/
mv simulation/* core/simulation/
mv tools/* core/tools/

# 3. 更新导入路径 (需要修改代码中的import语句)
# 例如: from agent.agent import DQNAgent → from core.agent.agent import DQNAgent

# 4. 创建 __init__.py 文件
touch core/__init__.py
touch core/agent/__init__.py
touch core/environment/__init__.py
touch core/simulation/__init__.py
# ... 等等
```

## 1. 前端 ↔ 后端 API 约定

### REST API

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|------|------|------|--------|------|
| POST | `/api/projects` | 创建项目 | `ProjectCreate` | `Project` |
| GET | `/api/projects` | 项目列表 | - | `Project[]` |
| GET | `/api/projects/{id}` | 项目详情 | - | `ProjectDetail` |
| PUT | `/api/projects/{id}/factory` | 保存工厂配置 | `FactoryConfig` | `FactoryConfig` |
| POST | `/api/optimization/start` | 启动优化 | `OptimizationParams` | `{task_id}` |
| POST | `/api/optimization/{task_id}/stop` | 停止优化 | - | `{status}` |
| GET | `/api/optimization/{task_id}/status` | 查询状态 | - | `TaskStatus` |
| GET | `/api/results/{project_id}/layouts` | 布局列表 | - | `LayoutSummary[]` |
| GET | `/api/results/layout/{layout_id}` | 布局详情 | - | `LayoutDetail` |
| POST | `/api/files/upload/cad` | 上传CAD | FormData | `{file_id, parsed_data}` |

### WebSocket

```
WS /api/ws/optimization/{task_id}

# 服务端推送消息格式:
{
  "type": "progress" | "layout" | "complete" | "error",
  "data": { ... }
```

## 2. 后端 ↔ 算法服务 接口约定

### OptimizationService

```python
class OptimizationService:
    def run(
        self,
        config: FactoryConfig,
        params: OptimizationParams,
        on_progress: Callable[[ProgressData], None],  # 进度回调
        on_layout: Callable[[LayoutData], None],      # 新布局回调
```

## 3. 核心数据结构

```python
# === 工厂配置 ===
class FunctionalUnit:
    id: str
    name: str
    width: float
    height: float
    type: str  # "station" | "dock" | "storage" | "obstacle"
    
class MaterialFlow:
    source: str
    target: str
    intensity: float

class FactoryConfig:
    width: float
    height: float
    units: list[FunctionalUnit]
    flows: list[MaterialFlow]
    fixed_positions: dict[str, Position]  # 固定位置的单元

# === 优化参数 ===
class OptimizationParams:
    project_id: str
    total_steps: int = 50000
    weights: ObjectiveWeights
    placement_order: str = "size_desc"

class ObjectiveWeights:
    distance: float = 0.25
    logistics: float = 0.25
    flow: float = 0.15
    throughput: float = 0.20
    utilization: float = 0.15

# === 进度/结果 ===
class ProgressData:
    episode: int
    total_episodes: int
    reward: float
    metrics: dict

class LayoutData:
    episode: int
    placements: dict[str, Position]  # unit_id -> {x, y, angle}
    reward: float
    metrics: Metrics

class Metrics:
    distance_score: float
    logistics_score: float
    throughput: float
    utilization: float
    flow_clarity: float
```
